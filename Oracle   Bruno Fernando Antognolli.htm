Olá pessoal, bom dia.

No post anterior mostrei como desabilitar os relacionamentos em SQL Server, hoje gostaria de compartilhar com vocês um script que faz o mesmo trabalho em Oracle.

Obs: Este aqui também desabilita triggers.

--SET DEFINE OFF;
 
--prompt desabilita TRIGGERS
-- DESABILITA TRIGGERS
BEGIN
 FOR db_cursor IN
 (SELECT TRIGGER_NAME, TABLE_NAME from USER_Triggers WHERE STATUS = 'ENABLED')
 LOOP
 BEGIN
 EXECUTE IMMEDIATE ('ALTER TRIGGER '||db_cursor.trigger_name ||' DISABLE' );
 END;
 END LOOP;
END;
/
--prompt desabilita fks
-- DESABILITANDO TODAS AS PKS E FKS
BEGIN
 FOR CURSOR_FK IN
 (SELECT CONSTRAINT_NAME, TABLE_NAME,CONSTRAINT_TYPE FROM USER_CONSTRAINTS
 WHERE CONSTRAINT_TYPE IN ('P','R','U')
 AND TABLE_NAME NOT LIKE '%$%'
 ORDER BY CONSTRAINT_TYPE DESC)
 LOOP
 BEGIN
 EXECUTE IMMEDIATE ('ALTER TABLE '||CURSOR_FK.TABLE_NAME||' DISABLE CONSTRAINT ' ||CURSOR_FK.CONSTRAINT_NAME);
 END;
 END LOOP;
END;
/
--DROPA INDEX UNICOS
BEGIN
 FOR C IN (SELECT * FROM USER_INDEXES I WHERE I.UNIQUENESS = 'UNIQUE')
 LOOP
 BEGIN
 EXECUTE IMMEDIATE ('DROP INDEX ' || C.INDEX_NAME);
 EXCEPTION WHEN OTHERS THEN
 DBMS_OUTPUT.put_line('ERRO INDEX: ' || C.INDEX_NAME || ' ' || SQLERRM);
 END;
 END LOOP;
END;

Copiar DBF Oracle

Publicado em 14 de janeiro de 2014 por Capitão Caverna
Resposta

Senhores, bom dia.

Um colega de trabalho apagou, acidentalmente, as entradas do registro do windows referentes ao Oracle.

Ele apagou todo o conteúdo dos registros do Oracle, localizado em:
HKEY_LOCAL_MACHINE/SOFTWARE/ORACLE

Pois bem, ao iniciar meu ambiente, notei que nenhum serviço do Oracle estava ativo (para ser mais exato, os serviços do Oracle nem são listados pelo Windows).

Então fui no painel de Controle “Adicionar/Remover Programa” que no Windows 8 está localizado em:
Painel de Controle\Todos os Itens do Painel de Controle\Programas e Recursos

E notei que o Windows não reconhece o Oracle como um programa instalado. Tentei subir o Oracle através do Prompt e não consegui.

Foi necessário instalar o Oracle do zero. (Reinstalar não era possível, visto que o Windows não o reconhecia como programa instalado). Antes eu possuía a instancia “ORCL” e com a nova instalação eu criei a “ORCL2″.

No diretório “E:\app\Bruno\oradata” existem duas pastas: “orcl” e “orcl2″.
E no diretório “E:\APP\BRUNO\FLASH_RECOVERY_AREA” também existem duas pastas: “orcl” e “orcl2″.

Eu copiei todo o conteúdo de “orcl” para “orcl2″ nos dois diretórios citados e depois fui para o SQL PLUS para subir o banco.

Detalhe: Iniciei o SQLPLUS a partir da nova instação do Oracle (pois eu mantive a pasta DBHOME antiga)

connect / as sysdba
 
SQL> shutdown immediate;
SQL> startup mount;
--verifica se o banco foi montado
SQL> select open_mode from v$database;
--Altera ORCL2 para ORCL
SQL> alter system set db_name=ORCL scope=spfile;
--Abre database
SQL> alter database open;

E com isso eu consegui voltar meu ambiente ao normal. Foi um pouco demorado, mas resolveu.

*OBS: Durante esse processo vocês poderão encontrar os seguintes erros: ORA-01122, ORA-01110, ORA-01206, ORA-01103, ORA-00211, ORA-00202. Todos foram resolvidos com o passo a passo acima.

*OBS2: Esse processo serve para mover databases entre computadores diferentes, com instâncias diferentes também.

Espero que tenha ajudado.

Pessoal, bom dia.

Uma dica rápida: Como configurar formatos data, hora, separador numérico na ferramenta PLSQL DEVELOPER?

PROBLEMAS ORACLE TNSLISTENER

Publicado em 1 de novembro de 2013 por Capitão Caverna
Resposta

Pessoal,

Estava utilizando uma máquina e eu estava com erro de TNS: ORA-12154.

Verifiquei os arquivos tnsnames.ora, listener.ora, revirei de baixo pra cima, até que eu simplesmente tentei testar o TNS local. Abri o CMD e digitei:

TNSPING NOME_DA_MINHA_INSTANCIA

E para a minha surpresa, nem o TNS local eu conseguia acessar. Como resolver esse problema?

Depois de um tempo eu fui verificar o serviço de Windows. Abra o executar, digite services.msc.

Procure pelo serviço: OracleOraDb11g_home1TNSListener

Se ele não estiver ativo, inicio-o e tente novamente o comando de TNSPING, provavelmente funcionará… algum espírito de porco deve ter desativado para testes e não voltou ao normal. Esse tipo de coisa acontece, infelizmente!

Abraços,

Problema Directory Oracle

Publicado em 1 de novembro de 2013 por Capitão Caverna
Resposta

Pessoal, boa noite.

Eu possuía uma tabela que lia um arquivo txt no meu servidor, para isso é necessário criar um directory no Oracle apontando para o caminho de seus arquivos txt. Existia uma procedure que posteriormente utilizava essa tabela e processava os registros. Até aqui, ok.

Fiz todo esse trabalho, criei minha tabela, e continuei meu desenvolvimento.  Em determinado momento precisei recriar o banco de dados e os todos os objetos, para a minha surpresa, quando eu criava uma transação anônima eu conseguia fazer um select na tabela, porém quando a minha procedure chamava a tabela durante o processamento, dava erro.

Ok, depois de pesquisar, descobri que o banco cria referência dos meus objetos (no caso minha procedure) para o “directory” criado no Oracle, quando eu dropei e subi o banco as referências de minha procedure estavam inválidas.

Solução?

Deletar e criar novamente o diretório do Oracle.

Após fazer isso, tanto minhas transações anônimas quanto meus objetos, conseguiam processar a minha tabela, que por sua vez, lia um arquivo txt.

Bom, espero ter ajudado.

Att,

Olá pessoal, boa noite.

Bom, uma dica útil hoje. Sabe aquele momento em que você pensa: “Puts, eu escrevi um  insert na tabela X, em alguma procedure ou trigger, mas não me lembro qual….”

Caso você tenha uma ferramenta, ela pode te ajudar. Porém se você não dispor de nada além de um SQLPLUS, você pode usar os objetos USER_SOURCE e USER_VIEWS. Como?

Select *
From USER_SOURCE
where UPPER(text) LIKE '%ADICIONE_SUA_PESQUISA_AQUI%'

select * from user_views

Expdp Impdp Oracle

Publicado em 8 de julho de 2013 por Capitão Caverna
Resposta

Pessoal, boa tarde.

Hoje vou compartilhar como realizar um import/export usando o data pump do Oracle.

Primeiramente, é interessante você saber os diretórios configurados no seu Oracle, para isso use:

1
2
--Descobrir qual o directory cadastrado no Oracle para realizar a exportação/importação
select * from all_directories

Em seguida, acesse o diretório em questão e execute o seguinte comando

1
2
--exportar pelo expdp
expdp USUARIO/SENHA@INSTANCIA schemas=ESQUEMAS_A_EXPORTAR directory=DATA_PUMP_DIR dumpfile=NOME_DUMP.dmp logfile=NOME_LOG.log

Para importar use o seguinte comando.

1
2
--importar usando o impdp
impdp USUARIO/SENHA@INSTANCIA remap_schema=SCHEMA:SCHEMA directory=DATA_PUMP_DIR dumpfile=NOME_DUMP.dmp logfile=NOME_LOG.log

Dica rápida. Esses dias precisei controlar transação dentro de uma Trigger e não lembrava como. Sei que existe o PRAGMA_TRANSACTION, mas não lembrava como usar.

Após algumas pesquisas, consegui o que desejava. Segue meu código:

CREATE OR REPLACE TRIGGER TESTE_TR_PRAGMA
BEFORE INSERT OR DELETE OR UPDATE ON TESTEPRAGMA
DECLARE
 PRAGMA AUTONOMOUS_TRANSACTION;
resultado_divisao number;

Pl Sql Developer Initialization error could not load oci.dll

Publicado em 4 de junho de 2013 por Capitão Caverna
2

Olá,

Estava montando meu ambiente de desenvolvimento Oracle no Windows 8 / 64 bits e tive problema com o SQLDeveloper, que mostrei como resolver aqui.

Além do SqlDeveloper eu costumo utilizar o PL SQL Developer e como nem tudo são flores, eu também tive problemas com a instalação do PL SQL Developer.

A mensagem que eu recebia ao tentar utilizar o PL SQL Developer dizia o seguinte:

_____________________________________________________________

[Window Title] PL/SQL Developer

[Content]
Initialization error
Could not initialize “C:\app\Bruno\product\11.2.0\dbhome_1\bin\oci.dll”
Make sure you have the 32 bits Oracle Client installed.

OracleHomeKey:
OracleHomeDir: C:\app\Bruno\product\11.2.0\dbhome_1
Found: oci.dll
Using: C:\app\Bruno\product\11.2.0\dbhome_1\bin\oci.dll
LoadLibrary(C:\app\Bruno\product\11.2.0\dbhome_1\bin\oci.dll) returned 0

[OK]
_____________________________________________________________

Para resolver esse problema primeiramente baixe um client Oracle de 32 Bits:

instantclient-basic-nt-11.2.0.3.0.zip
http://www.oracle.com/technetwork/topics/winsoft-085727.html

Em seguida, descompacte o client 32 bits em um diretório de sua preferência.

Na tela de login do PL Sql Developer clique em “CANCEL”, a tela inicial do programa será exibida. Procure por: Tolls -> Preferences.

Em “OCI Library” adicione o caminho da oci.dll 32bits.

Em seguida, o erro de DLL será resolvido. Aplique a alteração e reinicie sua ferramenta.

Bruno F. Antognolli.
